# SpiderMind 跨设备部署指南 v2.4

> 适用于任何 Windows 10/11 电脑，最快 3 分钟完成部署

---

## 一、部署前准备

### 你需要准备：

| 序号 | 准备项 | 说明 |
|------|--------|------|
| 1 | **Windows 电脑** | Windows 10 或 Windows 11 |
| 2 | **Python 3.9+** | 如未安装，下方有安装指南 |
| 3 | **Coze API Token** | 免费申请，下方有获取教程 |
| 4 | **网络连接** | 首次安装依赖需要联网 |

## 二、详细部署步骤

### 步骤 1：安装 Python（如已安装可跳过）

1. 访问 Python 官网：https://www.python.org/downloads/
2. 点击黄色按钮下载最新版 Python
3. **重要：** 安装时**务必勾选底部的 "Add Python to PATH"**，然后点 "Install Now"
4. 验证安装：打开 CMD（Win+R，输入 cmd，回车），输入 `python --version`，显示版本号即成功

> 如果上述方法失败，也可以用 Microsoft Store 搜索 "Python 3.12" 安装

### 步骤 2：解压 SpiderMind 压缩包

1. 将 `SpiderMind_v2.4_xxx.zip` 解压到**任意位置**（建议 `C:\SpiderMind\` 或桌面）
2. 解压后文件夹结构：
   ```
   SpiderMind/
   ├── main.py              ← 主程序
   ├── setup.bat            ← 首次运行：安装依赖
   ├── 启动SpiderMind.bat    ← 日常启动：一键运行
   ├── .env.example         ← Token 配置模板
   ├── requirements.txt     ← Python 依赖列表
   ├── coze_api.py          ← AI 接口模块
   ├── token_manager.py     ← Token 管理
   ├── components/          ← 界面组件
   └── pages/               ← 各功能页面
   ```

### 步骤 3：首次运行 — 双击 `setup.bat`

1. 双击 `setup.bat`
2. 脚本会自动：
   - 检测 Python 环境
   - 创建虚拟环境（隔离依赖）
   - 安装所需依赖（streamlit、pandas、requests 等）
   - 从 `.env.example` 自动创建 `.env` 配置文件
3. 等待 1-3 分钟，直到显示"安装完成"

### 步骤 4：配置 Coze API Token

1. 用记事本打开 `SpiderMind` 文件夹中的 `.env` 文件
2. 找到这一行：`COZE_API_TOKEN=在这里填入你的Token`
3. 将其替换为你的真实 Token

#### 如何获取 Token：
1. 浏览器访问：https://www.coze.cn
2. 使用微信/手机号登录
3. 点击右上角**头像** → **个人中心**
4. 左侧菜单选择**「访问令牌」**
5. 点击**「新建令牌」**按钮
6. **过期时间：选择「无过期时间」**（避免答辩时过期）
7. 权限勾选：`chat`
8. 点击「确认」，**立即复制显示的 Token**
9. 粘贴到 `.env` 文件中的 `COZE_API_TOKEN=` 后面
10. 保存 `.env` 文件

### 步骤 5：启动系统 — 双击 `启动SpiderMind.bat`

1. 双击 `启动SpiderMind.bat`
2. 脚本会自动：
   - 检测 Python 和依赖
   - 验证 Token 配置
   - 智能选择可用端口（自动避开占用）
   - 启动 Streamlit 服务
   - 打开浏览器访问系统
3. 浏览器自动打开 `http://localhost:8502`（端口可能自动调整为 8503/8504 等）

## 三、常见问题解决

### Q1：双击 bat 文件闪退
**原因：** Python 未安装或未添加到 PATH

**解决：**
1. 打开 CMD（Win+R → 输入 cmd → 回车）
2. 输入 `python --version`
3. 如果提示"不是内部命令"，说明 Python 未正确安装
4. 重新安装 Python，**安装时务必勾选 "Add Python to PATH"**
5. 安装完成后重启电脑

### Q2：setup.bat 提示"依赖安装失败"
**原因：** 网络问题或 pip 源连接超时

**解决（选择一种）：**
- 方法一：关闭 VPN/代理后重试
- 方法二：打开 CMD，进入 SpiderMind 目录，运行：
  ```
  pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```

### Q3：启动后显示 "Token 未配置" 或 "Token 无效"
**原因：** .env 文件未配置或 Token 已过期

**解决：**
1. 删除 `.env` 文件
2. 重新双击 `setup.bat`（会自动生成新的 .env 模板）
3. 按步骤 4 重新获取并填入 Token

### Q4：浏览器打开后页面空白
**原因：** Streamlit 正在加载，需要等待几秒

**解决：**
- 等待 5-10 秒，刷新页面
- 如果仍空白，检查 CMD 窗口是否有红色错误提示

### Q5：端口 8502 被占用
**解决：**
- 系统已内置自动端口检测，会自动切换到 8503/8504 等
- 查看 CMD 窗口显示的最终端口号，手动在浏览器输入对应地址

### Q6：健康问答 / AI 解读无响应
**原因：** Coze API 连接问题

**解决：**
1. 确认网络可以访问 `https://www.coze.cn`
2. 检查 .env 中的 Token 是否正确
3. Token 格式应为 `pat_` 开头的一长串字符

### Q7：在其他电脑解压后中文文件名乱码
**原因：** 压缩工具编码问题

**解决：**
- 使用 Windows 自带的"提取全部"功能解压
- 或使用 7-Zip / WinRAR 等工具

## 六、安全提醒

- ⚠️ `.env` 文件中包含你的 Token，**不要分享给他人**
- ⚠️ 压缩包中的 `.env.example` 是模板，不包含真实 Token
- ⚠️ 所有健康数据存储在本地 `data/` 目录，不上传云端
